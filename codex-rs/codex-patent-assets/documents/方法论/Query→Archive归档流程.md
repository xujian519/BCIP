# Query→Archive 归档流程设计

> 基于 Karpathy LLM Wiki 方法论：好的问答结果应该沉淀为新的 wiki 页面，不应消失在聊天历史中。

---

## 核心原则

当用户对知识库提出查询，Claude 综合多个 wiki 页面给出答案时，如果该答案具有**独立参考价值**，应归档为新 wiki 页面。

## 归档判断标准

满足以下**任意两条**即可归档：

| 条件 | 说明 | 示例 |
|------|------|------|
| 跨源综合 | 答案综合了3个以上不同来源的知识库页面 | 比较以案说法、崔国斌、审查指南对"公知常识"的不同标准 |
| 新分析 | 答案产生了原始页面中不存在的比较、对比或联系 | 对比创造性判断在化学领域和电学领域的实务差异 |
| 高复用性 | 该问题可能被多次查询 | "实用新型专利创造性判断标准" |
| 决策支持 | 答案提供了具体的实务操作指引 | "撰写化学领域权利要求时的常见驳回理由和应对策略" |

## 归档流程

```
用户提问 → Claude搜索wiki → 综合答案 → 判断是否值得归档
                                                    ↓ 是
                                            创建wiki页面（标记来源）
                                                    ↓
                                            更新相关index.md
                                                    ↓
                                            在log.md中记录
```

## 页面模板

```markdown
# {主题标题}

> **页面类型：** 问答归档
> **生成日期：** {YYYY-MM-DD}
> **触发问题：** "{原始问题}"
> **综合来源：** [[页面A]]、[[页面B]]、[[页面C]]
> **关联页面：** [[{相关主题页面}}]]

---

## 核心结论

{2-3句话直接回答原始问题}

---

## 详细分析

{综合多个来源的分析内容}

---

## 实务建议

{具体的操作指引}

---

## 关联知识

- **原始来源页面：** [[页面A]]、[[页面B]]
- **相关主题：** [[相关页面]]
```

## 命名规则

- 归档页面命名：`{主题}-综合-{子主题}.md`
- 放入最相关的知识库目录下
- 文件名前缀 `综合-` 表示这是跨源综合页面

## 与 Ingest 的区别

| | Ingest（摄取） | Query→Archive（归档） |
|---|---|---|
| 触发 | 新 raw 文件到达 | 用户提问 |
| 来源 | 单一 raw 文件 | 多个已有 wiki 页面 |
| 页面类型 | 来源摘要页 | 跨源综合页 |
| 标记 | `来源：《xxx》` | `页面类型：问答归档` |

## 实现方式

在 Claude Code 的 CLAUDE.md 中添加以下指令：

```markdown
## Query→Archive 规则

当用户提出问题且你的答案综合了3个以上 wiki 页面时：
1. 先正常回答用户问题
2. 如果答案满足归档标准，主动询问："这个分析值得归档为 wiki 页面吗？"
3. 用户确认后，按归档模板创建页面，更新 index.md 和 log.md
```
