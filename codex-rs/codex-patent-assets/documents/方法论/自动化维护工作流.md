# 自动化维护工作流

页面类型：方法论

本文档记录知识库的自动化维护工具和日常工作流，确保知识库质量持续可维护。

---

## 日常维护节奏

| 频率 | 操作 | 命令 |
|------|------|------|
| 每次编译新页面后 | 运行 lint 检查 | `python3 lint_wiki.py` |
| 每周 | 增量更新图谱 | Claude Code 中运行 `/graphify ./Wiki --update` |
| 每月 | 全量重建图谱 + 社区导读 | `/graphify ./Wiki` 然后 `/graphify ./Wiki --wiki` |
| 按需 | 查询知识盲区 | `/graphify ./Wiki --query "主题关键词"` |

## Git Hook 说明

### pre-commit hook

知识库已配置 pre-commit hook，在每次 `git commit` 时自动运行 `lint_wiki.py`。

**行为逻辑：**
1. 检测暂存区是否有 `Wiki/*.md` 文件变更
2. 如果没有 Wiki 文件变更，直接跳过
3. 如果有，运行 `lint_wiki.py` 进行全库健康检查
4. 如果发现 **ERROR**（如断链、缺失 index.md），阻止提交
5. 如果只有 **WARNING**（如孤儿页面、过大页面），允许提交
6. 检查通过后正常提交

### 如何跳过检查

```bash
git commit --no-verify -m "你的提交信息"
```

### 如何禁用 hook

```bash
# 临时禁用（删除 hook 文件）
rm .git/hooks/pre-commit

# 重新安装（从仓库根目录）
# 重新创建 .git/hooks/pre-commit 并 chmod +x
```

### 如何重新安装 hook

如果 hook 文件丢失，参考 `.git/hooks/pre-commit` 的内容重新创建，确保：
- 文件路径：`.git/hooks/pre-commit`
- 有可执行权限：`chmod +x .git/hooks/pre-commit`

## 质量保障流程

### lint 检查项说明

`lint_wiki.py` 对每个知识库执行以下检查：

| 检查项 | 级别 | 说明 |
|--------|------|------|
| index.md 存在 | ERROR | 每个知识库必须有索引文件 |
| log.md 存在 | WARN | 建议有编译日志 |
| 断链检查 | ERROR | `[[链接]]` 指向不存在的页面 |
| 虚拟链接 | WARN | 指向约定命名但未创建的页面（如审查指南条款引用） |
| 页面大小 > 8000字 | WARN | 建议拆分为更小的页面 |
| 核心摘要缺失 | WARN | 页面缺少 `## 核心要点` 等摘要标题 |
| 孤儿页面 | WARN | 没有被任何其他页面链接的页面 |

### Graphify 图谱报告解读

运行 graphify 后，在 `graphify-out/` 目录生成：

| 文件 | 说明 |
|------|------|
| `graph.html` | 交互式知识图谱可视化 |
| `graph.json` | 图谱数据（节点和边） |
| `GRAPH_REPORT.md` | 图谱分析报告 |
| `manifest.json` | 文件处理清单（用于增量更新） |

**如何解读报告：**
- **社区聚类**：自动识别的知识主题群组，检查是否符合预期的知识结构
- **桥接节点**：连接多个社区的关键页面，通常是核心概念页面
- **孤立节点**：未被图谱连接的页面，可能是孤儿页面或缺少交叉引用
- **密度指标**：社区内部连接密度，密度过低说明该主题的页面之间缺乏关联

### 利用 Graphify 发现知识盲区

1. 查看图谱中的**稀疏区域** — 页面少但应该有内容的主题
2. 检查**桥接节点** — 是否有重要概念缺少独立页面
3. 对比**社区大小** — 某个主题特别小的可能需要补充
4. 查询特定主题的关联 — `/graphify ./Wiki --query "主题"` 查看覆盖程度

## 四层架构维护规则

知识库采用四层架构，每层有不同的维护规则：

### 第一层：Raw/ — 不可变原始材料

- **绝对不修改** Raw/ 下的任何文件
- 新的原始资料直接添加到对应子目录
- OCR 质量问题记录在案但不修改原文

### 第二层：Wiki/ — 编译产出

- 所有页面由 LLM 编译生成，遵循各知识库的 CLAUDE.md 规范
- **lint 保障质量**：每次提交自动检查，确保基本结构完整
- **交叉引用网络**：四大学说源（审查指南、以案说法、崔国斌、尹新天）互相链接
- **多源一致性标记**：跨源对比时使用 `⟷一致`、`⟷侧重差异`、`⟷标准分歧`、`⟷时代差异`

### 第三层：brainstorming/ — 探索沉淀

- 个人探索和讨论的草稿
- 不纳入 lint 检查范围
- 定期将有价值的内容归档到 Wiki/（通过 Query→Archive 流程）
- 归档参考：[[Query→Archive归档流程]]

### 第四层：artifacts/ — 最终产出

- 基于知识库生成的最终文档
- 如分析报告、法律意见书模板等
- 依赖 Wiki/ 层的内容，不依赖 Raw/ 直接引用
